# Роутер
- Предназначен для маршрутизации пакетов
- Связь идет от роутера к роутеру
- Имеет мало портов, поэтому не следует ставить его в центр сети, а нужно использовать скорее как gateway

## Программное обеспечение маршрутизаторов Cisco
- Все маршрутизаторы работают под управлением операционной системы Cisco IOS
- Для каждой модели существует несколько разновидностей операцонной системы
- Образы IOS различаются по версии

Память
В маршрутизаторах как правило устанавливается 4 типа памяти
- ROM (неизменяемое ПЗУ)
    - Содержит POST (Power-On Self Test)
    - Программу ROM Monitor
    - Сокращенная версия IOS, которая используется для загрузки основной версии
- Flash (изменяемое ПЗУ)
    - Содержит образ операционной системы IOS (до 32 Мб)
- RAM (оперативная память). Делится на два типа
    - Процессорная память
    - Память ввода-вывода
        - Помещается в маршрутизирумые пакеты
- NVRAM (Non-Volatile Random-Access Memory, изменяемое ПЗУ) - т.е. энергонезависимая память
    - Содержит конфигурацию маршрузитатора (до 64 Кб)

## Интерфейсы маршрутизатора
Каждый маршрутизатор имеет несколько физических интерфейсов

Наиболее распространенными типами интерфейсов являются
- Ethernet, Fast Ethernet
- Последовательный (Serial)
    - Бывают синхронный, синхронно-асинхронный и асинхронный
- Виртуальный интерфейс
    - Null, Loopback

### Свойства интерфейса loopback
- Этот интерфейс всегда активен
- Пакеты, направленные на интерфейс, считаются отправленными маршрутизатору
- Пакеты, маршрутизированные через такой интерфейс, уничтожаются
    - Применяется при создании фильтрации датаграм и создании защитных маршрутов

## Информация о работе маршрутизатора
### Интерфейс командной строки
Маршрутизатор конфигурируется в командной строке операционной системы Cisco IOS.
Присоединение к маршрутизатору осуществляется через Telnet любого из его интерфейсов. 

Аварийное отключение от консоли не регистрируется маршрутизатором (сеанс останется в том же состоянии)

При работе в командной строке Cisco IOS существует несколько контекстов (режимов ввода команд)

Контекст пользователя открывается при подсоединии к маршрутизатору

В контексте пользователя доступны только простые команды, не влияющие на конфигурацию маршрутизатора.

Контекст администратора открывается командой `enable`
Контекст администратора позволяет получить полную конфигурацию о маршрутизаторе и его состоянии

`Router#`

Обратный переход в контекст пользователя происходит по команде `disable`

`exit` - выход на уровень выше

`config` - конфигурировать через терминал



Глобальный контекст содержит как команды конфигурирования маршрутизатора, так и команды перехода в контексты конфигурирования подсистем


```
router(config)#  глобальный
router(config-if)#  интерфейсы
router(config-router)#  динамический маршрутизации
router(config-line)#  терминальная линии
```





Каждому интерфейсу соответствует разъем маршрутизатора

Маршрутизаторы также имеют консольные интерфейсы (CON и AUX)
    - CON подключается к компьютеру администратора
    - AUX для удаленного доступа по телефонной линии
