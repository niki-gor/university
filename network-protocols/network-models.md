Фирма CISCO использует иерархическую межсетевую модель, которая представляет собой многоуровневое отображение топологического проекта объединенной сети

Модель разработана в целях максимального повышения производительности, масштабируемости и оптимальной отказоустройчивости 

Состоит из трех уровней
1. Уровень ядра - здесь происходит концентрация всего сетевого трафика. Необходима максимальная скорость передачи данных при минимальных затратах на маршрутизацию и коммутацию
2. Распределительный - маршрутизация, обработка списков доступа и обеспечение качества обслуживания. Шифрование, трансляция сетевых адресов
- Уровень доступа - в небольшие коммутируемые сети, построенные по технологии Ethernet

## Статическая маршрутизация и маршрутизация по умолчанию
### Статическая
Оператор сети вручную вводит пути передачи пакетов в таблицы маршрутизации

#### Достоинства
- В линиях связи отсутствует служебный трафик
    - Эффективность линии связи повышается
- Т.к. администратор устанавливает маршруты передачи данных к определенным сетям, то обеспечивается хорошая защита информации

#### Недостатки
- Неприменима в крупных сетях - слишком большой оверхед на обновление всех хостов

### Динамическая
Существует 3 класса протоколов динамической маршрутизации

#### Вектора расстояния (distance vector)
  - Протоколы для вектора расстояния используют для поиска наилучшего пути расстояния до удаленной сети


Хопом называется каждое перенаправление пакета маршрутизатором

Наилучшим считается путь к удаленной сети с наименьшим количеством хопов

Вектор (IP-адрес сети назначения) определяет направление к удаленной сети

К протоколам по вектору расстояния относятся:

- RIP (Routing Information Protocol)
- IGRP (Interior Gateway Routing Protocol) - проприетарный протокол CISCO
- BGP (Border Gateway Protocol)
- EIGRP (Ehanced IGRP) - гибридный свойства дистанционно-векторных и link-state

#### Link State
Первым выбирается кратчайший путь

Каждый маршрутизатор создает 3 отдельные таблицы:
- одна отслеживает подключенных соседей
- вторая определяет топологию всей объединенной сети
- таблица маршрутизации

Протоколы
- OSPF (Open Shortest Path First)
- IS-IS (Intermediate System to Intermediate System)
- NLSP (Network Link Service Protocol)

#### Гибридные
- EIGRP (Ehanced IGRP) - гибридный свойства дистанционно-векторных и link-state

## Фильтрация трафика с помощью списков доступа (ACL)
Набор инструкций, которые применяются к интерфейсу маршрутизатора.
Эти инструкции указывают маршрутизатору, какие пакеты следует принять, а какие отвергнуть.

Решения основываются на определенных критериях
- Адрес источника
- Адрес получателя
- Номер порта
- и т.д.

При составлении списков управления доступов важен порядок, в котором указываются соответствующие директивы.

При фильтрации трафик используют следующие типы списков доступа
1. Стандартные нумерованные и именованные
2. Расширенные нумерованные и именованные
3. Рефлексивные
